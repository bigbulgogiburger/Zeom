## 3. 핵심 기능 추가

### 3.1 [P0] 무료 콘텐츠 퍼널 — 사용자 유입 엔진

> **벤치마크**: 점신 (일일 운세 보고서), Co-Star (매일 푸시 운세)
>
> 현재 운세 페이지가 존재하나 비로그인 시 블러 처리만 됨. 무료 콘텐츠가 없으면 신규 유저 유입 경로가 극히 제한적.

**구현 방안:**

```
[비로그인 사용자]
  ├── 오늘의 운세 미리보기 (총운 점수 + 1줄 요약 무료 공개)
  ├── 띠별 운세 (12개 모두 무료)
  ├── 간단 궁합 테스트 (생년월일만으로 즉시 결과)
  └── 블로그/콘텐츠 (사주 기초 상식, 용어 해설)

[로그인 사용자]
  ├── 전체 운세 보고서 (카테고리별 상세 점수)
  ├── AI 맞춤 코멘트 (감정 흐름, 대인 관계)
  ├── 대운/세운 차트 시각화
  └── 상담사 1:1 유료 상담 유도 CTA
```

**핵심 지표**: 비로그인 → 로그인 전환율 목표 15%+

### 3.2 [P0] 상담사 프로필 고도화

> **벤치마크**: 사주나루 (상담사 1,000명, 상세 프로필), 숨고 (전문가 인증)

**현재**: `name`, `specialty`, `intro`, `supportedConsultationTypes`만 존재

### 3.3 [P0] 실시간 상담 (채팅 상담 모드)

> **벤치마크**: 천명 (채팅+전화), Talkspace (세션 간 메시징), 소울톡 (채팅 타로)

**현재**: 화상 상담(Sendbird Calls)만 지원. 채팅 상담 인프라(`ChatController`, `ChatService`, `ChatMessageEntity`)는 구현되어 있으나 프론트에서 제대로 활용되지 않음.

**구현 방안:**
- 예약 없이 즉시 채팅 상담 시작 가능 (크레딧 차감 방식)
- 상담 중 사진/파일 첨부 기능
- AI 상담 노트 자동 생성 (상담 종료 후 요약본 제공)
- 채팅 ↔ 화상 전환 기능 (상담 중 "화상으로 전환" 버튼)

### 3.4 [P0] 첫 상담 무료 체험

> **벤치마크**: 점신 (첫 3분 무료), BetterHelp (첫 주 할인)

**현재**: 무료 체험 메커니즘 없음. 신규 유저의 첫 결제 허들이 높음.

**구현 방안:**
- 회원가입 시 무료 크레딧 3분 자동 지급
- 첫 상담 3분 경과 후 "계속 상담하시겠어요?" 모달 + 자동 결제 연동
- 쿠폰 시스템 연계 (이미 `coupon/` 패키지 존재)

### 3.5 [P1] 상담사-고객 매칭 알고리즘

> **벤치마크**: BetterHelp (상세 인테이크 → 정밀 매칭), 점신 (AI 추천)

**현재**: 온보딩에서 관심사/고민을 수집하지만 실제 매칭에 반영되지 않음 (`loadRecommendations`가 단순히 전체 목록에서 3개를 자르는 방식).

**개선:**
```
매칭 점수 = (전문분야 일치 × 0.3)
          + (평균 평점 × 0.25)
          + (응답률 × 0.2)
          + (가격 범위 일치 × 0.15)
          + (온라인 상태 × 0.1)
```
- 온보딩 데이터(생년월일, 관심분야, 고민유형)를 `user_preferences` 테이블에 저장
- 상담 이력 기반 재추천 로직

### 3.6 [P1] 소셜 기능

> **벤치마크**: Co-Star (친구 궁합), 점신 (공유 카드)

- **궁합 기능**: 두 사람의 사주 궁합 점수 + 상세 분석
- **공유 카드**: 오늘의 운세를 SNS 공유 가능한 이미지 카드로 생성
- **초대 리워드**: 친구 초대 시 양쪽 모두 무료 크레딧 지급 (이미 `referral/` 패키지 존재)

### 3.7 [P1] 상담 후 경험 강화

> **벤치마크**: 천명 (AI 상담노트), The Pattern (생애 주기 타이밍)

- **상담 요약 리포트**: 상담 종료 후 AI가 핵심 포인트 정리 → 앱에서 언제든 열람
- **후속 알림**: "상담 후 7일" 체크인 알림 → 재상담 유도

**개선안:**
- **서버 상태**: React Query (TanStack Query) 도입 → 캐싱, 리페칭, 낙관적 업데이트

**현재 이슈:**

### 5.5 [P0] 보안 강화

**즉시 수정 필요:**
- **JWT를 httpOnly cookie로 전환**: 현재 localStorage 저장 → XSS 공격 시 토큰 탈취 가능. `Set-Cookie: HttpOnly; Secure; SameSite=Strict`로 변경
- **deviceId UUID 생성**: 웹에서 `'web-main'` 하드코딩 → `crypto.randomUUID()` + localStorage 저장

**추가 강화:**
- CSRF 토큰 적용 (httpOnly cookie 전환 시 필수)
- Rate limiting 세분화 (로그인 시도, 결제 API에 더 엄격한 제한)
- 입력 값 서버 사이드 검증 강화 (Bean Validation 어노테이션 전수 확인)
- 결제 금액 서버 사이드 재검증 (클라이언트 전달 금액 신뢰 금지)
- 감사 로그 확대 (결제, 환불, 분쟁 관련 모든 액션 로깅)
- N+1 쿼리 점검: Entity fetch join 명시적 추가 (슬롯, 크레딧 조회)

### 5.6 [P2] 인프라/DevOps

- Docker Compose 개발 환경 (Backend + Redis + MySQL 원클릭 실행)

- 에러 트래킹 (Sentry 통합 — 웹 + Flutter)
