## 1. 시스템 개요

천지연꽃신당의 운세 엔진은 **사주팔자(四柱八字)** 이론을 프로그래밍으로 구현한 시스템입니다.

사용자의 생년월일시를 입력받아 4개의 기둥(년주·월주·일주·시주)을 계산하고, 매일의 일진(日辰)과의 상호작용을 분석하여 **개인화된 일일 운세**를 생성합니다.

### 두 가지 운세 모드

| 모드 | 대상 | 방식 | 정확도 |
|------|------|------|--------|
| **사주 기반** | 생시 정보를 입력한 사용자 | 만세력 알고리즘 + 십성/12운성/합충 분석 | 높음 (전통 사주 풀이 기반) |
| **SHA-256 폴백** | 생시 미입력 사용자 | `SHA-256(userId + date)` 해시 기반 | 랜덤 (결정론적이지만 의미 없음) |

### 외부 의존성

**없음.** 순수 Java로 구현되어 있으며, Spring Framework 외에 외부 라이브러리를 사용하지 않습니다. 천문 데이터, 만세력 테이블 모두 코드 내에 내장되어 있습니다.

### 3.1 천간 (天干, Heavenly Stems) — 10개

|------|------|------|------|------|
| 1 | 갑 | 甲 | 목(木) | 양(+) |
| 2 | 을 | 乙 | 목(木) | 음(-) |
| 3 | 병 | 丙 | 화(火) | 양(+) |
| 4 | 정 | 丁 | 화(火) | 음(-) |
| 5 | 무 | 戊 | 토(土) | 양(+) |
| 6 | 기 | 己 | 토(土) | 음(-) |
| 7 | 경 | 庚 | 금(金) | 양(+) |
| 8 | 신 | 辛 | 금(金) | 음(-) |

| 3 | 인 | 寅 | 목 | 양 | 호랑이 | 03:30~05:30 |
| 4 | 묘 | 卯 | 목 | 음 | 토끼 | 05:30~07:30 |
| 5 | 진 | 辰 | 토 | 양 | 용 | 07:30~09:30 |
| 6 | 사 | 巳 | 화 | 음 | 뱀 | 09:30~11:30 |

| 9 | 신 | 申 | 금 | 양 | 원숭이 | 15:30~17:30 |

### 3.3 오행 (五行, Five Elements)

만물을 구성하는 5가지 원소와 그 관계입니다.

```
        상생(생성) 순환                    상극(파괴) 순환
        ──────────────                    ──────────────

          木 (목)                            木 (목)
         ↗     ↘                           ↗     ↘
       水        火                       金        土
       (수)      (화)                     (금)      (토)
         ↖     ↙                           ↖     ↙
          金 ← 土                            火 ← 水
         (금)  (토)                         (화)  (수)

  목→화→토→금→수→목                  목→토→수→화→금→목
  (나무가 불을 만들고,                (나무가 흙을 뚫고,
   불이 재를 만들고...)                흙이 물을 막고...)
```

**두 오행 간 5가지 관계**:

| 관계 | 의미 | 예시 |
|------|------|------|
| **비화** | 같은 오행 | 목 ↔ 목 |
| **상생** | 내가 생해주는 | 목 → 화 |
| **피생** | 나를 생해주는 | 화 → 목 |
| **상극** | 내가 극하는 | 목 → 토 |
| **피극** | 나를 극하는 | 금 → 목 |

> **구현**: `OhaengEnum.java` — `getSangsaeng()`, `getSanggeuk()`, `getRelation(other)` 메서드

### 3.4 십성 (十星/十神, Ten Gods)

일간(日干, 나 자신)을 기준으로 다른 천간과의 관계를 분류한 10가지 유형입니다. **운세 텍스트의 핵심 결정 요소**입니다.

| 십성 | 한자 | 판정 조건 | 키워드 |
|------|------|-----------|--------|
| 비견 | 比肩 | 같은 오행 + 같은 음양 | 경쟁, 자립, 독립심 |
| 겁재 | 劫財 | 같은 오행 + 다른 음양 | 손재, 분쟁, 도전 |
| 식신 | 食神 | 내가 생하는 + 같은 음양 | 재능, 표현, 풍요 |
| 상관 | 傷官 | 내가 생하는 + 다른 음양 | 창의, 반항, 변화 |
| 편재 | 偏財 | 내가 극하는 + 같은 음양 | 투자, 횡재, 사교 |
| 정재 | 正財 | 내가 극하는 + 다른 음양 | 안정 수입, 절약 |
| 편관 | 偏官 | 나를 극하는 + 같은 음양 | 권위, 압박, 도전 |
| 정관 | 正官 | 나를 극하는 + 다른 음양 | 명예, 규율, 관직 |
| 편인 | 偏印 | 나를 생하는 + 같은 음양 | 학문, 변화, 비상 |
| 정인 | 正印 | 나를 생하는 + 다른 음양 | 지원, 안정, 학업 |

**판정 알고리즘** (`SipseongEnum.calculate(dayGan, targetGan)`):

```
1. 일간의 오행(A)과 대상의 오행(B) 비교
2. 음양 일치 여부 확인

if A == B           → 비견 (같은 음양) / 겁재 (다른 음양)
if A가 B를 생      → 식신 (같은 음양) / 상관 (다른 음양)
if A가 B를 극      → 편재 (같은 음양) / 정재 (다른 음양)
if B가 A를 극      → 편관 (같은 음양) / 정관 (다른 음양)
if B가 A를 생      → 편인 (같은 음양) / 정인 (다른 음양)
```

## 4. 사주 명식 계산 엔진

사주팔자란 **4개의 기둥(柱)**으로 구성됩니다. 각 기둥은 천간 1개 + 지지 1개 = 2글자이므로, 총 **8글자(八字)**입니다.

```
┌────────┬────────┬────────┬────────┐
│  년주  │  월주  │  일주  │  시주  │
│ (年柱) │ (月柱) │ (日柱) │ (時柱) │
├────────┼────────┼────────┼────────┤
│ 천간   │ 천간   │ 천간   │ 천간   │  ← 하늘의 기운
│ 지지   │ 지지   │ 지지   │ 지지   │  ← 땅의 기운
├────────┼────────┼────────┼────────┤
│ 조상   │ 부모   │ 나자신 │ 자녀   │  ← 대표하는 관계
│ 사회운 │ 직업운 │ 성격   │ 말년운 │  ← 대표하는 운
└────────┴────────┴────────┴────────┘
```

> **가장 중요한 글자**: 일간(日干) = 일주의 천간 = **"나 자신"**
> 모든 운세 판단은 일간을 기준으로 다른 글자들과의 관계를 분석합니다.

### 4.1 년주 (年柱) 계산

> **파일**: `SajuCalculator.calculateYearPillar()`

**핵심 규칙**: 사주에서 한 해의 시작은 1월 1일이 아니라 **입춘(立春, ~2월 4일)**입니다.

```
알고리즘:
1. 입춘 이전이면 year -= 1 (전년도 기준)
2. 천간 = (year - 3) % 10   → CheonganEnum 인덱스
3. 지지 = (year - 3) % 12   → JijiEnum 인덱스
```

**예시**: 1990년 3월 15일생
- 입춘(2/4) 이후 → year = 1990
- 천간: (1990-3) % 10 = 7 → **경(庚, 금, 양)**
- 지지: (1990-3) % 12 = 3 → **인(寅, 목, 양)**
- **년주 = 경인(庚寅)**

### 4.2 월주 (月柱) 계산

> **파일**: `SajuCalculator.calculateMonthPillar()` + `JeolgiData.java`

**두 단계로 계산합니다**:

#### Step A: 월지(月支) — 24절기 기준

사주의 월은 양력 월이 아니라 **절기(節氣)** 기준입니다:

| 절기 | 양력 대략 | 사주 월 | 월지 |
|------|-----------|---------|------|
| 입춘 | 2/4 | 1월 | 인(寅) |
| 경칩 | 3/6 | 2월 | 묘(卯) |
| 청명 | 4/5 | 3월 | 진(辰) |
| 입하 | 5/6 | 4월 | 사(巳) |
| 망종 | 6/6 | 5월 | 오(午) |
| 소서 | 7/7 | 6월 | 미(未) |
| 입추 | 8/7 | 7월 | 신(申) |
| 백로 | 9/8 | 8월 | 유(酉) |
| 한로 | 10/8 | 9월 | 술(戌) |
| 입동 | 11/7 | 10월 | 해(亥) |
| 대설 | 12/7 | 11월 | 자(子) |
| 소한 | 1/6 | 12월 | 축(丑) |

#### Step B: 월간(月干) — 오호결원법(五虎遁元法)

년간에 따라 **1월(인월)의 시작 천간**이 결정됩니다:

| 년간 | 1월(인월) 시작 천간 |
|------|---------------------|
| 갑(甲) / 기(己) | **병(丙)** (index 3) |
| 을(乙) / 경(庚) | **무(戊)** (index 5) |
| 병(丙) / 신(辛) | **경(庚)** (index 7) |
| 정(丁) / 임(壬) | **임(壬)** (index 9) |
| 무(戊) / 계(癸) | **갑(甲)** (index 1) |

```
월간 = 시작천간인덱스 + (사주월번호 - 1)
```

**예시**: 1990년 3월 15일생
- 3/15은 경칩(3/6) 이후 → 사주 2월(묘월)
- 월지: 묘(卯, index 4)
- 년간이 경(庚) → 시작 천간 = 무(戊, index 5)
- 월간: 5 + (2-1) = 6 → **기(己)**  ← 실제로는 오호결원법에서 경→무이고, 5+(2-1)=6 → 기
- 하지만 실제 코드에서: 을/경 → start=5(무), monthGan = 5 + 1 = 6 → 기...

아, 재확인하면: 년간=경(庚, index 7), 경은 을/경 그룹 → startGan=5(무), sajuMonth=2, monthGanIndex = 5 + (2-1) = 6 → 기(己)
- 하지만 인월부터 시작이므로 2월은 인덱스 1 차이
- **월주 = 기묘(己卯)**

실제 전통 만세력에서 확인하면: 1990년 경오월 전후를 봐야 하는데, 코드의 오호결원법 구현에 따르면 위와 같습니다.

### 4.3 일주 (日柱) 계산 — 만세력 알고리즘

> **파일**: `ManseryeokData.java`

일주는 **60갑자(甲子~癸亥)가 무한 반복하는 고정 패턴**이므로, 기준일로부터의 일수 차이만 알면 계산 가능합니다.

```
기준일: 2000년 1월 1일 = 갑진(甲辰)일
  → 천간 기준 인덱스 = 1 (甲)
  → 지지 기준 인덱스 = 5 (辰)

계산:
  days = 대상일 - 2000-01-01 (일수 차이)
  일간 인덱스 = (days % 10 + 기준간인덱스) [1~10 순환]
  일지 인덱스 = (days % 12 + 기준지인덱스) [1~12 순환]
```

> **O(1) 연산** — 어떤 날짜든 나눗셈 한 번으로 즉시 계산

### 4.4 시주 (時柱) 계산 — 오자결원법

> **파일**: `SajuCalculator.calculateHourPillar()`

#### Step A: 시지(時支) — 12시진 매핑

출생 시간을 12시진으로 변환합니다 (사용자가 회원가입 시 선택):

```
자시(子時) = 23:30~01:30
축시(丑時) = 01:30~03:30
...
해시(亥時) = 21:30~23:30
```

#### Step B: 시간(時干) — 오자결원법(五子遁元法)

일간에 따라 **자시(子時)의 시작 천간**이 결정됩니다:

```
일간 그룹화: dx = (일간인덱스 > 5) ? 일간인덱스 - 5 : 일간인덱스

시간 인덱스 = 시지인덱스 + ((dx - 1) × 2)
```

**생시를 모르는 경우**: `hourPillar = null` — 시주 없이 3기둥만으로 운세 산출 (정확도 감소)

### 4.5 오행 균형 분석

> **파일**: `SajuChart.java`

4기둥의 8글자(천간 4개 + 지지 4개)에서 각 오행이 몇 번 등장하는지 세어 **오행 분포**를 계산합니다:

```java
// 각 기둥의 천간·지지의 오행을 카운트
countPillar(balance, yearPillar);   // 2글자
countPillar(balance, monthPillar);  // 2글자
countPillar(balance, dayPillar);    // 2글자
countPillar(balance, hourPillar);   // 2글자 (없으면 skip)
```

결과 예시: `{목: 2, 화: 3, 토: 0, 금: 2, 수: 1}`

- **우세 오행(dominant)**: 가장 많은 원소 → 성격의 주요 특성
- **부족 오행(weak)**: 가장 적은 원소 → 보완이 필요한 영역

## 5. 일일 운세 스코어링 알고리즘

> **파일**: `DailyFortuneCalculator.java`

매일 아침, 사용자의 사주 명식과 **오늘의 일진(日辰)**을 비교하여 점수를 산출합니다.

### 5.1 점수 공식

```
총운 점수 = 기본점수(50)
           + 12운성 가중치          (-8 ~ +10)
           + 오행 관계 보너스        (-10 ~ +15)
           + 합충형파해 보정         (-15 ~ +20)

→ 최종 범위: 1 ~ 100으로 클램프
```

### 5.2 12운성 가중치

일간(나)이 오늘의 지지를 만났을 때의 에너지 상태:

```java
TwelveUnseongEnum unseong = TwelveUnseongEnum.calculate(myDayGan, todayJi);
score += unseong.getScoreWeight();  // -8 ~ +10
```

예: 일간이 임(壬)이고 오늘 지지가 사(巳) → **쇠(衰, -2점)**

### 5.3 오행 관계 보너스

일간의 오행과 오늘 천간의 오행 관계:

| 관계 | 점수 보정 | 의미 |
|------|-----------|------|
| 상생 (내가 생) | +5 | 에너지 발산 |
| 피생 (나를 생) | **+15** | 도움 받음, 최고 |
| 비화 (같은 오행) | +8 | 동지, 협력 |
| 상극 (내가 극) | -5 | 에너지 소모 |
| 피극 (나를 극) | **-10** | 압박, 최악 |

### 5.4 합충형파해(合沖刑破害) 보정

오늘의 지지가 사주 차트의 지지들과 맺는 특수 관계:

| 관계 | 조건 | 점수 | 의미 |
|------|------|------|------|
| **삼합(三合)** | 3개 지지가 삼각형 | **+20** | 대길, 큰 행운 |
| **육합(六合)** | 특정 2개 지지 조합 | **+10** | 조화, 좋은 인연 |
| **충(沖)** | 인덱스 차이 = 6 | **-15** | 충돌, 갈등 |
| **형(刑)** | 특정 형벌 조합 | **-10** | 시련, 법적 문제 |
| **파(破)** | 특정 파괴 조합 | **-8** | 파손, 계획 차질 |
| **해(害)** | 특정 해 조합 | **-5** | 질투, 방해 |

**충(沖) 계산 예시**:

```
자(子, 1) ↔ 오(午, 7)  — 차이 6 → 충!
축(丑, 2) ↔ 미(未, 8)  — 차이 6 → 충!
인(寅, 3) ↔ 신(申, 9)  — 차이 6 → 충!
```

### 5.5 카테고리별 점수

총운 점수를 기반으로 카테고리별 조정:

| 카테고리 | 조정 범위 | 기반 |
|----------|-----------|------|
| 재물운 | -8 ~ +10 | 십성이 재성(편재/정재)이면 보너스 |
| 애정운 | -5 ~ +8 | 십성이 인성(편인/정인)이면 보너스 |
| 건강운 | -8 ~ +5 | 12운성이 병(病)/사(死)이면 감점 |
| 직업운 | -5 ~ +10 | 십성이 관성(편관/정관)이면 보너스 |
| 학업운 | -5 ~ +8 | 십성이 인성이면 보너스 |

### 5.6 행운 아이템 생성

| 항목 | 결정 방식 |
|------|-----------|
| 행운의 색 | 일간의 상생 오행 색상 (목→청록, 화→적색, 토→황색, 금→백색, 수→흑색) |

| 행운의 시간 | 상생 오행의 시간대 (목시간: 인시/묘시 등) |
| 주의 시간 | 상극 오행의 시간대 |

## 9. SHA-256 폴백 시스템

> **파일**: `FortuneService.generateHashFortune()`

생시 정보를 입력하지 않은 사용자를 위한 폴백입니다.

```
해시 = SHA-256("userId=42&date=2026-02-25")
  → 32바이트 (256비트)

바이트 매핑:
  bytes[0]~[3]  → 총운 점수     (1~100)
  bytes[4]~[7]  → 재물운 점수
  bytes[8]~[11] → 애정운 점수
  bytes[12]~[15]→ 건강운 점수
  bytes[16]     → 행운의 색 인덱스
  bytes[17]     → 행운의 숫자
  bytes[18]     → 행운의 방향 인덱스
```

**특성**:
- 같은 사용자 + 같은 날 = 항상 같은 결과 (결정론적)
- 사주 이론과 무관 (의미 없는 랜덤)
- 텍스트는 `FortuneTexts.java`의 일반 운세 텍스트 사용

## 10. 실제 계산 예시

### 입력

```
이름: 홍길동
생년월일: 1990년 3월 15일 (양력)
생시: 오시 (午時, 11:30~13:30)
성별: 남
오늘 날짜: 2026년 2월 25일
```

### Step 1: 사주 명식 계산

```
① 년주: 입춘(2/4) 이후 → 1990년 기준
   천간: (1990-3) % 10 = 7 → 경(庚, 금, 양)
   지지: (1990-3) % 12 = 3 → 인(寅, 목, 양)
   → 년주: 경인(庚寅)

② 월주: 3/15 → 경칩(3/6) 이후 → 사주 2월(묘월)
   월지: 묘(卯, index 4)
   오호결원법: 경 → 시작천간 무(5)
   월간: 5 + (2-1) = 6 → 기(己)
   → 월주: 기묘(己卯)

③ 일주: 2000-01-01부터의 일수 차이
   1990-03-15 → -3579일 (2000년 기준 과거)
   간: ((-3579 % 10 + 10) % 10 + 1) = 계산...
   (실제 만세력 기반 계산)
   → 일주: 임오(壬午) [예시]

④ 시주: 오시(午) + 오자결원법
   일간 임(壬, index 9) → dx = 9-5 = 4
   시간인덱스: 7 + ((4-1)×2) = 7+6 = 13 → 13-10 = 3 → 병(丙)
   → 시주: 병오(丙午)
```

### 완성된 명식

```
┌────────┬────────┬────────┬────────┐
│  경(庚) │  기(己) │  임(壬) │  병(丙) │  ← 천간
│  인(寅) │  묘(卯) │  오(午) │  오(午) │  ← 지지
├────────┼────────┼────────┼────────┤
│  년주   │  월주   │  일주   │  시주   │
│  금+목  │  토+목  │  수+화  │  화+화  │  ← 오행
└────────┴────────┴────────┴────────┘

오행 분포: 목=2, 화=3, 토=1, 금=1, 수=1
우세: 화(火) | 부족: 토/금/수 (동률)
```

### Step 2: 오늘의 운세 스코어링

```
오늘(2026-02-25)의 일진 계산:
  → 가정: 기사(己巳)일

▸ 나의 일간: 임(壬, 수)
▸ 오늘 천간: 기(己, 토)
▸ 오늘 지지: 사(巳, 화)

① 12운성: 임(壬) + 사(巳) → 조견표 → 절(絶, -8점)

② 오행 관계: 수(임) vs 토(기)
   토극수 → "피극" → -10점

③ 십성: 임(수) vs 기(토)
   토가 수를 극함 → "나를 극하는"
   임=양, 기=음 → 다른 음양
   → 정관(正官) ← 명예, 규율

④ 합충형파해:
   오늘 사(巳, 6) vs 내 지지들:
   - 인(寅, 3): 인사(寅巳) → 형(刑) → -10점
   - 오(午, 7): 사오(巳午) → 없음
   - 묘(卯, 4): 없음

⑤ 총점 계산:
   기본      50
   12운성    -8  (절)
   오행관계  -10  (피극)
   형        -10  (인사형)
   ──────────────
   합계      22점 → 클램프(1,100) → 22점

⑥ 등급: 22점 → 1등급(나쁨)
```

### Step 3: 최종 운세 출력

```
총운: 22/100 — "외부의 압박이나 예상치 못한 도전에 직면할 수 있습니다..."
재물: 32/100 — 정관 보정 → "체계적인 재무 계획이 필요한 날입니다"
애정: 28/100 — "관계에서 오해가 생기기 쉬우니 대화에 신중하세요"
건강: 15/100 — 절(絶) → "건강 관리에 각별히 주의하세요"

행운의 색: 흑색 (수의 상생=목의 색? 아니면 나를 생하는 금→백색)
행운의 숫자: (9+6) % 9 + 1 = 7
행운의 방향: 북쪽 (수의 방위)
행운의 시간: 신시 (15:30~17:30, 금 시간 = 나를 생하는)
```
